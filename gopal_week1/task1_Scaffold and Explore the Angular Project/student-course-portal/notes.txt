Angular Project File Explanations - student-course-portal
=============================================================

1. angular.json - Main Angular CLI configuration file that defines build, serve, and project architecture settings including output paths, assets, and production budgets.

2. tsconfig.json - TypeScript compiler configuration that sets global compiler options like target ES version, module resolution, and strict type checking rules for the entire project.

3. tsconfig.app.json - Application-specific TypeScript configuration that extends tsconfig.json and specifies the src/ directory files to compile while excluding test files.

4. package.json - Node.js package manifest file listing project metadata, npm scripts (build, serve, test), and dependencies required for Angular and the application.

5. src/main.ts - Application entry point that bootstraps the root Angular component (App) with the provided configuration and handles any startup errors.

6. src/app/app.config.ts - Defines Angular providers and application configuration for the standalone root component, including routing and error listeners.

7. src/app/app.ts - Root component (standalone mode) that serves as the main application component with router outlet and application title signal.

8. src/index.html - Entry HTML file that loads the Angular application inside the <app-root> element and provides metadata like viewport settings and page title.

Budget Configuration Notes from angular.json:
==============================================
The budgets configuration under architect > build > configurations > production controls:
- "initial" type with maximumWarning: 500kB and maximumError: 1MB - Limits the size of the main initial bundle
- "anyComponentStyle" type with maximumWarning: 4kB and maximumError: 8kB - Limits individual component stylesheet sizes
These thresholds trigger warnings during build if exceeded and errors if maximumError is breached, helping maintain optimal application performance.

Build Output (ng build) - dist/ Folder Structure:
=================================================
The ng build command generates the following structure in dist/student-course-portal/:
- browser/ folder contains the compiled application:
  * main-[hash].js - The compiled application code (214.57 kB raw size, 58.89 kB gzipped)
  * index.html - The entry HTML file
  * styles-[hash].css - The compiled CSS styles
  * favicon.ico - The application icon
- 3rdpartylicenses.txt - Third-party package licenses
- prerendered-routes.json - Routing configuration metadata

The main-[hash].js file contains the complete compiled Angular application code, including all components, services, and dependencies bundled into a single JavaScript bundle.
